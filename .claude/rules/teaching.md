# 学習スタイルのルール

## コマンドの提示方法

- 実行するコマンドはユーザーが自身で打つ。Claude は提示のみ行い、自分では実行しない
- ただし git コマンドは Claude が代わりに実行する
- コマンドはシンタックスハイライト付きのコードブロックで提示する

## コミットのタイミング

- 区切りの良いタイミングで自己判断でコミットする（ユーザーへの確認不要）

## コードの記述

- Claude がコードを全て提示し、ユーザーがそれを写経する形で学習する
- コードを提示する前に、必ず「何のために・なぜこのファイルを書くのか」を説明する
- コードを提示する際は、各部分の意図・役割を説明する

## 説明スタイル

- 新しい概念を導入するときは、JavaScript/フロントエンドの知識と対応づけて説明する
- コマンドの意図と効果を簡潔に補足する
